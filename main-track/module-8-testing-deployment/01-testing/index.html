    Testing with Postman¶

    ภาพรวม¶

    Postman เป็นเครื่องมือสำหรับทดสอบ API โดยไม่ต้องสร้าง Frontend

    🎯 สิ่งที่จะได้เรียนรู้¶

    • ติดตั้งและใช้งาน Postman
    • ทดสอบ GET/POST Request
    • จัดการ Cookies และ Sessions
    • สร้าง Collection ทดสอบ

    📦 ติดตั้ง Postman¶

    1. ดาวน์โหลดจาก postman.com/downloads
    2. ติดตั้งและเปิดโปรแกรม
    3. สมัคร account (ฟรี) หรือ Skip

    🔵 ทดสอบ GET Request¶

    ดึงสินค้าทั้งหมด¶

    1. คลิก New → HTTP Request
    2. เลือก Method = GET
    3. ใส่ URL: 
      http://localhost/vshop/backend/api/products/get-all.php
    4. คลิก Send

    ผลลัพธ์ที่คาดหวัง¶

    {
    "status": 200,
    "message": "ดึงข้อมูลสำเร็จ",
    "data": {
        "products": [
            {
                "id": 1,
                "name": "iPhone 15 Pro Max",
                "price": 52900,
                "stock": 25
            }
        ]
    }
}

    🟢 ทดสอบ POST Request¶

    Login¶

    1. เลือก Method = POST
    2. ใส่ URL: 
      http://localhost/vshop/backend/api/auth/login.php
    3. Tab Headers: เพิ่ม 
      Content-Type: application/json
    4. Tab Body: เลือก raw และ JSON
    5. ใส่ JSON:
    {
    "email": "admin@vshop.com",
    "password": "admin123"
}
    1. คลิก Send

    ผลลัพธ์ที่คาดหวัง¶

    {
    "status": 200,
    "message": "เข้าสู่ระบบสำเร็จ",
    "data": {
        "user": {
            "id": 1,
            "name": "Admin",
            "email": "admin@vshop.com",
            "role": "admin"
        }
    }
}

    🍪 จัดการ Cookies¶

    ดู Session Cookie¶

    หลังจาก Login สำเร็จ:

    1. คลิก Cookies (ใต้ปุ่ม Send)
    2. จะเห็น PHPSESSID=xxxxxxxx
    3. Postman จะส่ง cookie นี้ไปในทุก request โดยอัตโนมัติ
    sequenceDiagram
    participant P as 📮 Postman
    participant API as ⚙️ PHP API
    participant S as 🔐 Session

    P->>API: POST /login { email, password }
    API->>S: session_start()
    API->>S: $_SESSION['user'] = user
    API-->>P: Set-Cookie: PHPSESSID=xxx
    Note over P: เก็บ cookie

    P->>API: GET /cart (Cookie: PHPSESSID=xxx)
    API->>S: session_start()
    API->>S: ดึง $_SESSION['user']
    API-->>P: { cart items }

    📁 สร้าง Collection¶

    จัดกลุ่ม requests ให้เป็นระเบียบ:

    สร้าง Collection ใหม่¶

    1. คลิก Collections (ซ้ายมือ)
    2. คลิก + → Blank collection
    3. ตั้งชื่อ "V-Shop API"

    จัดกลุ่ม Requests¶

    V-Shop API/
├── Auth/
│   ├── Login
│   ├── Register
│   ├── Logout
│   └── Get Me
├── Products/
│   ├── Get All
│   ├── Get One
│   ├── Create (Admin)
│   ├── Update (Admin)
│   └── Delete (Admin)
└── Cart/
    ├── Get Cart
    ├── Add Item
    └── Remove Item

    🧪 Test Scripts¶

    Postman สามารถทดสอบอัตโนมัติได้:

    Tab Tests¶

    // ตรวจสอบ status code
pm.test("Status code is 200", function () {
    pm.response.to.have.status(200);
});

// ตรวจสอบ response body
pm.test("Response has products", function () {
    var jsonData = pm.response.json();
    pm.expect(jsonData.data.products).to.be.an('array');
});

// ตรวจสอบเวลา response
pm.test("Response time is less than 500ms", function () {
    pm.expect(pm.response.responseTime).to.be.below(500);
});

    ✅ Checklist ทดสอบ API¶

    Auth APIs¶

    Request Method Expected Status
    Login (correct) POST 200
    Login (wrong password) POST 401
    Register (new user) POST 201
    Register (duplicate email) POST 409
    Get Me (logged in) GET 200
    Get Me (not logged in) GET 401
    Logout POST 200

    Products APIs¶

    Request Method Expected Status
    Get All Products GET 200
    Get Product by ID GET 200
    Get Product (not found) GET 404
    Create Product (admin) POST 201
    Create Product (not admin) POST 403
    Update Product PUT 200
    Delete Product DELETE 200

    Cart APIs¶

    Request Method Expected Status
    Get Cart (logged in) GET 200
    Get Cart (not logged in) GET 401
    Add to Cart POST 201
    Remove from Cart DELETE 200
