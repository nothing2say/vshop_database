    ระบบ Login¶

    สิ่งที่จะได้เรียนรู้¶

    • สร้าง Login API
    • ใช้ password_verify()
    • จัดการ Session

    Login ทำงานอย่างไร?¶

    ┌──────────────┐                    ┌──────────────┐                    ┌──────────────┐
│   Frontend   │  POST /login       │   Backend    │  SELECT user       │   Database   │
│              │ ─────────────────► │              │ ─────────────────► │              │
│  email       │                    │  ตรวจสอบ     │                    │  users       │
│  password    │ ◄───────────────── │  password    │ ◄───────────────── │              │
└──────────────┘  JSON Response     └──────────────┘  User data         └──────────────┘

    ขั้นตอนการ Login¶

    1. รับ email และ password จาก Frontend
    2. ค้นหา user จาก email
    3. ตรวจสอบ password ด้วย password_verify()
    4. ถ้าถูกต้อง → สร้าง session และส่งข้อมูล user กลับ
    5. ถ้าผิด → ส่ง error message

    password_verify() คืออะไร?¶

    password_verify() เปรียบเทียบ password ที่ผู้ใช้กรอก กับ hash ใน database

    <?php
// password ที่ hash เก็บไว้ใน database
$hashedPassword = '$2y$10$abc...xyz';

// password ที่ผู้ใช้กรอก
$inputPassword = 'password123';

// ตรวจสอบ
if (password_verify($inputPassword, $hashedPassword)) {
    echo "รหัสผ่านถูกต้อง!";
} else {
    echo "รหัสผ่านไม่ถูกต้อง";
}
?>

    ห้ามเปรียบเทียบตรงๆ

    // ❌ ผิด - ไม่มีทาง match กัน
if ($inputPassword === $hashedPassword) { ... }

// ✅ ถูก - ใช้ password_verify()
if (password_verify($inputPassword, $hashedPassword)) { ... }

    สร้างไฟล์ login.php¶

    โครงสร้างไฟล์¶

    backend-mysql/
└── api/
    └── auth/
        └── login.php    ← สร้างไฟล์นี้

    โค้ดสมบูรณ์¶

    <?php
/**
 * Login API
 * POST /api/auth/login
 *
 * Input: { "email": "...", "password": "..." }
 * Output: { "status": 200, "message": "...", "data": {...} }
 */

require_once '../../config/database.php';

// ต้องเป็น POST เท่านั้น
if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
    sendResponse(405, 'Method not allowed. Use POST.');
}

// รับข้อมูล JSON
$input = getJSONInput();

$email = sanitizeInput($input['email'] ?? '');
$password = $input['password'] ?? '';

// ===================================
// Validation
// ===================================

if (empty($email)) {
    sendResponse(400, 'Email is required');
}

if (empty($password)) {
    sendResponse(400, 'Password is required');
}

if (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
    sendResponse(400, 'Invalid email format');
}

// ===================================
// ค้นหา User
// ===================================

try {
    $stmt = $pdo->prepare("
        SELECT id, email, password, name, role
        FROM users
        WHERE email = ?
    ");
    $stmt->execute([$email]);
    $user = $stmt->fetch();

    // ไม่พบ user
    if (!$user) {
        sendResponse(401, 'Invalid email or password');
    }

    // ===================================
    // ตรวจสอบ Password
    // ===================================

    if (!password_verify($password, $user['password'])) {
        sendResponse(401, 'Invalid email or password');
    }

    // ===================================
    // สร้าง Session
    // ===================================

    session_start();
    $_SESSION['user_id'] = $user['id'];
    $_SESSION['user_email'] = $user['email'];
    $_SESSION['user_name'] = $user['name'];
    $_SESSION['user_role'] = $user['role'];

    // ส่งข้อมูล user กลับ (ไม่รวม password)
    unset($user['password']);

    sendResponse(200, 'Login successful', [
        'user' => $user
    ]);

} catch (PDOException $e) {
    sendResponse(500, 'Database error: ' . $e->getMessage());
}
?>

    อธิบายโค้ด¶

    1. ตรวจสอบ Method¶

    if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
    sendResponse(405, 'Method not allowed. Use POST.');
}

    Login ต้องใช้ POST เพราะส่ง password ไม่ควรใช้ GET (จะแสดงใน URL)

    2. รับและ Validate ข้อมูล¶

    $email = sanitizeInput($input['email'] ?? '');
$password = $input['password'] ?? '';

if (empty($email)) {
    sendResponse(400, 'Email is required');
}

    Password ไม่ต้อง sanitize

    Password ไม่ต้อง sanitize เพราะจะนำไป hash เปรียบเทียบเท่านั้น

    3. ค้นหา User¶

    $stmt = $pdo->prepare("
    SELECT id, email, password, name, role
    FROM users
    WHERE email = ?
");
$stmt->execute([$email]);
$user = $stmt->fetch();

    ใช้ Prepared Statement ป้องกัน SQL Injection

    4. ตรวจสอบ Password¶

    if (!password_verify($password, $user['password'])) {
    sendResponse(401, 'Invalid email or password');
}

    Error Message ไม่ควรบอกชัดเจน

    ไม่ควรบอกว่า "Email not found" หรือ "Wrong password" เพราะ hacker จะรู้ว่า email ไหนมีอยู่จริง

    5. สร้าง Session¶

    session_start();
$_SESSION['user_id'] = $user['id'];
$_SESSION['user_email'] = $user['email'];
$_SESSION['user_name'] = $user['name'];
$_SESSION['user_role'] = $user['role'];

    เก็บข้อมูล user ไว้ใน session สำหรับใช้ในหน้าอื่น

    ทดสอบด้วย Postman¶

    Request¶

    • Method: POST
    • URL: http://localhost/vshop/backend-mysql/api/auth/login.php
    • Headers: Content-Type: application/json
    • Body (raw JSON):
    {
    "email": "admin@vshop.com",
    "password": "admin123"
}

    Response (สำเร็จ)¶

    {
    "status": 200,
    "message": "Login successful",
    "data": {
        "user": {
            "id": 1,
            "email": "admin@vshop.com",
            "name": "Admin",
            "role": "admin"
        }
    }
}

    Response (ผิดพลาด)¶

    {
    "status": 401,
    "message": "Invalid email or password"
}

    Error Cases¶

    สถานการณ์ Status Message
    ไม่ใส่ email 400 Email is required
    ไม่ใส่ password 400 Password is required
    Email format ผิด 400 Invalid email format
    ไม่พบ user 401 Invalid email or password
    Password ผิด 401 Invalid email or password
    Database error 500 Database error: ...
