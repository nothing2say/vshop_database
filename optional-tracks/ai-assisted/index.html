    AI-Assisted Coding¶

    ภาพรวม¶

    AI สามารถช่วยเขียนโค้ด ออกแบบ Database และแก้ปัญหาได้

    เครื่องมือที่แนะนำ¶

    Tool ข้อดี ราคา
    Claude เข้าใจ context ดี, คำตอบละเอียด Free / Pro
    ChatGPT เร็ว, หลากหลาย Free / Plus
    GitHub Copilot เขียนโค้ดใน Editor $10/เดือน

    ใช้ AI ช่วยอะไรได้บ้าง?¶

    1. ออกแบบ Database¶

    ตัวอย่าง Prompt:

    ช่วยออกแบบ Database สำหรับระบบร้านขายสินค้าออนไลน์ที่มี:
- สินค้าและหมวดหมู่
- ลูกค้าและ admin
- ตะกร้าสินค้า
- การสั่งซื้อ

ให้แสดง:
1. ER Diagram
2. SQL สร้างตาราง
3. อธิบายความสัมพันธ์

    2. เขียนโค้ด¶

    ตัวอย่าง Prompt:

    เขียน PHP API สำหรับ login ที่:
- รับ email และ password เป็น JSON
- ใช้ PDO เชื่อมต่อ MySQL
- ใช้ password_verify() ตรวจสอบ
- สร้าง session เมื่อ login สำเร็จ
- ส่ง JSON response
- ป้องกัน SQL Injection

Database มีตาราง users (id, email, password, name, role)

    3. Debug ปัญหา¶

    ตัวอย่าง Prompt:

    โค้ด PHP นี้มี error:

[วาง error message]
[วาง code]

ช่วยอธิบายว่า:
1. สาเหตุของ error คืออะไร
2. แก้ไขอย่างไร
3. วิธีป้องกันในอนาคต

    4. อธิบายโค้ด¶

    ตัวอย่าง Prompt:

    อธิบายโค้ด PHP นี้:

[วาง code]

อธิบาย:
1. ทำอะไรทีละบรรทัด
2. ใช้ function อะไรบ้าง
3. Flow การทำงาน

    5. เรียนรู้ Concept¶

    ตัวอย่าง Prompt:

    อธิบาย SQL Injection ให้เข้าใจง่าย:
- มันคืออะไร
- เกิดขึ้นได้อย่างไร
- ยกตัวอย่างโค้ดที่มีช่องโหว่
- วิธีป้องกัน

    เทคนิคเขียน Prompt ที่ดี¶

    โครงสร้าง Prompt ที่แนะนำ¶

    Context: [อธิบาย background/ระบบที่กำลังทำ]
Current State: [สถานะปัจจุบัน/สิ่งที่มีอยู่แล้ว]
Requirement: [สิ่งที่ต้องการ]
Constraints: [ข้อจำกัด - ภาษา, เทคโนโลยี]
Expected Output: [รูปแบบผลลัพธ์ที่ต้องการ]

    ตัวอย่างที่ดี¶

    Context: กำลังสร้างระบบ E-commerce ด้วย PHP + MySQL

Current State:
- มีตาราง users, products, cart แล้ว
- สร้าง login API เสร็จแล้ว

Requirement: สร้าง API เพิ่มสินค้าลงตะกร้า

Constraints:
- ใช้ PHP + PDO
- ต้อง login ก่อนเพิ่มได้
- ตรวจสอบ stock ก่อนเพิ่ม
- ถ้ามีในตะกร้าแล้วให้เพิ่มจำนวน

Expected Output:
- PHP code พร้อม comment ภาษาไทย
- ตัวอย่าง request/response

    ตัวอย่างที่ไม่ดี¶

    ❌ "เขียน API cart"
❌ "ช่วยเขียน PHP"
❌ "แก้ error นี้ให้หน่อย"

    ข้อควรระวัง¶

    1. ตรวจสอบความถูกต้อง¶

    AI อาจให้คำตอบที่: - ผิดพลาด - ล้าสมัย - ไม่เหมาะกับ context

    ต้องตรวจสอบและทดสอบเสมอ!

    2. เข้าใจก่อนใช้¶

    ❌ Copy โค้ดไปใช้โดยไม่เข้าใจ
✅ อ่าน ทำความเข้าใจ แล้วค่อยใช้

    3. ระวังข้อมูลลับ¶

    ❌ ส่ง API key, password, ข้อมูลลูกค้า ให้ AI
✅ แทนที่ด้วย placeholder ก่อนส่ง

    4. AI ไม่ได้รู้ทุกอย่าง¶

    • ข้อมูลอาจไม่อัพเดท
    • อาจไม่รู้ library เฉพาะทาง
    • Context ของโปรเจค AI ไม่รู้

    แหล่งเรียนรู้¶

    • Claude - AI จาก Anthropic
    • ChatGPT - AI จาก OpenAI
    • GitHub Copilot - AI ใน Editor
    • Prompt Engineering Guide - เทคนิคเขียน Prompt

    สรุป¶

    AI เป็นเครื่องมือที่ช่วยได้มาก แต่:

    • ต้องเข้าใจพื้นฐานก่อน
    • ต้องตรวจสอบความถูกต้อง
    • ใช้เป็นผู้ช่วย ไม่ใช่ทำแทนทั้งหมด
    • ยิ่งถาม Prompt ดี ยิ่งได้คำตอบดี
